Beyond `.note()` and `.offset()`, which we discussed in the previous section, instruments have a variety of methods for manipulating and transforming patterns.

## Pattern Setters

These methods set the pattern of the instrument directly. Each argument passed to the methods is a bar, and can be either a value or an array of values which define the number of steps in that bar.

These methods spread steps evenly in time across a bar, but the pattern modifier methods documented next can be used add variety and complexity to your bars.

### `note(...patterns)`

Used with **synthesizers** to set the frequency of pitches that should be played during a bar. By default, this method expects [MIDI note numbers](https://inspiredacoustics.com/en/MIDI_note_numbers_and_center_frequencies). However, it is often used with the `root` and `scale` methods to normalize and clamp the notes to a specific range.

Depending on whether a value, an array of values, or an array of array of values is passed in, this method can produce single notes, runs, or chords, respectively. Passing in falsey values like `null` or `undefined` create a rest.

| Argument    | Type                                   | Required |
| ----------- | -------------------------------------- | -------- |
| ...patterns | `number` \| `number[]` \| `number[][]` | `true`   |

```js
.note(60)                    // one bar, one note
.note([60, 64, 67, 71])      // one bar, four notes
.note([[60, 64, 67, 71]])    // one bar, one chord
.note([[60, 64], [67, 71]])  // one bar, two chords
```

### `offset(...patterns)`

Conceptually, this mathod functions similarly to the `note` method described above. Instead of expecting MIDI notes, though, it expects numbers between `0` and `1`, which defines the point in the sample where playback should begin. For example, passing in `0.5` will begin playing halfway through the duration of the sample.

| Argument    | Type                                   | Required |
| ----------- | -------------------------------------- | -------- |
| ...patterns | `number` \| `number[]` \| `number[][]` | `true`   |

```js
d.sample("my_loop").offset([0.25, 0, 0.625, 0.5]).push();
```

## Pattern Modifiers

### `xox(...patterns)`

Enables [X0X-style step sequencing](https://www.youtube.com/watch?v=YvOWB2UsbJs&t=55s), as made famouse by drum machines like the TR-808. You can define patterns in either of two ways: a string, using `x` to denote hits and any other character to denote rests; or an array, using `1` (or a truthy value) to denote hits and `0` (or a falsy value) to denote rests. Passing multiple strings or arrays creates a series of patterns

| Argument    | Type                   | Required |
| ----------- | ---------------------- | -------- |
| ...patterns | `string` \| `number[]` | `true`   |

d.sample("bd").xox("x . . . x . . .").push();
d.sample("hh").xox("x x x x x - x x").push();
d.sample("sd").xox([0, 0, 1, 0, 0, 0, 1, 0]).push();`}
/>

### `.euclid(pulses, steps, rotation)`

Generates a [Euclidean rhythm](https://en.wikipedia.org/wiki/Euclidean_rhythm) using an algorithm that evenly distributes a number of pulses across a specified number of steps — a classic technique for generating polyrhythmic patterns.

Passing an optional rotation argument shifts the pattern to the left (positive) or right (negative) by that many steps. Passing an array of values for either the pulses or rotation arguments creates multiple bars.

| Argument | Type                 | Required |
| -------- | -------------------- | -------- |
| pulses   | `number \| number[]` | `true`   |
| steps    | `number`             | `true`   |
| rotation | `number \| number[]` | `false`  |

<CodeEditor
  client:load
  code={`
// 3 hits over 8 steps, rotated 3 steps to the left
d.sample("bd:3").euclid(3,8,3).push();
// 2 hits over 8 steps, rotated 2 to the left 
d.sample("sd").euclid(2,8,2).gain(0.625).push(); 
// [11, 13] hits over 16 steps, rotated 5 steps to the right
d.sample("hh").euclid([11,13],16,-5).gain([0.5,0.25]).push();`}
/>

### hex

Express a pattern as a hexadecimal string or number. Each hex digit encodes 4 steps as a bitmask.

```js
d.sample("bd").hex("8888").push(); // "1000 1000 1000 1000" → kick on beats 1&3
d.sample("hh").hex("ffff").push(); // all 16 steps active
d.sample("sd").hex("0808").push(); // snare on beats 2&4
```

You can also pass numbers:

```js
d.sample("bd").hex(0x8080).push();
```

### sequence

Place specific pulses within a fixed number of steps.

```js
.sequence(steps, ...pulsePositions)
```

```js
d.sample("bd").sequence(8, 0, 4).push(); // 8 steps, hits at positions 0 and 4
d.sample("sd").sequence(8, [2, 6]).push(); // hits at positions 2 and 6
```

### arrange

Define a multi-bar pattern by specifying how many bars each sub-pattern repeats.

```js
.arrange([numBars, pattern], [numBars, pattern], ...)
```

```js
d.sample("bd")
  .arrange(
    [2, [1, null, null, null]], // play this for 2 bars
    [2, [1, null, 1, null]], // then this for 2 bars
  )
  .push();
```

### fast / slow

Speed up or slow down the pattern by a multiplier.

```js
d.sample("hh").note(1, 1, 1, 1).fast(2).push(); // plays twice as fast (8 hits per bar)
d.sample("bd").note(1, null, 1, null).slow(2).push(); // plays half as fast (spans 2 bars)
```

### stretch

Stretches the pattern across more steps, inserting silences between hits.

```js
d.sample("bd").note(1, null, 1, null).stretch(2).push();
// original: hit, rest, hit, rest
// stretched: hit, rest, rest, rest, hit, rest, rest, rest
```

### reverse

Reverses the pattern.

```js
d.synth("sine").root("C4").scale("major").note(0, 2, 4, 7).reverse().push();
// plays 7, 4, 2, 0
```

### legato

Sustains each note until the next one starts, instead of stopping at the end of its step.

```js
d.synth("sine").root("C4").note(0, null, 2, null).legato().push();
// the 0 sustains through the rest until the 2 plays
```

Pass `false` to turn legato off:

```js
.legato(false)
```
