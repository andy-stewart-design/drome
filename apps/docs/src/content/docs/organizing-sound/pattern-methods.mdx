Beyond `.note()` and `.offset()`, which we discussed in the previous section, instruments have a variety of methods for manipulating and transforming patterns.

## Pattern Setters

These methods set the pattern of the instrument directly. Each argument passed to the methods is a bar, and can be either a value or an array of values which define the number of steps in that bar.

These methods simply spread steps evenly in time across a bar, but the pattern modifier methods documented next can be used add variety and rhythmic complexity to your bars.

### `note(...patterns)`

Used with **synthesizers** to set the frequency of pitches that should be played during a bar. By default, this method expects [MIDI note numbers](https://inspiredacoustics.com/en/MIDI_note_numbers_and_center_frequencies). However, it is often used with the `root` and `scale` methods to normalize and clamp the notes to a specific range.

```js
.note(60)                    // one bar, one note
.note([60, 64, 67, 71])      // one bar, four notes
.note([[60, 64, 67, 71]])    // one bar, one chord
.note([[60, 64], [67, 71]])  // one bar, two chords
```

Depending on whether a value, an array of values, or a nested array of values is passed in, this method can produce single notes, runs/arpeggios, or chords, respectively. Passing in falsey values like `null` or `undefined` create a rest note.

| Argument    | Type                                   | Required |
| ----------- | -------------------------------------- | -------- |
| ...patterns | `number` \| `number[]` \| `number[][]` | `true`   |

<CodeEditor
  client:load
  code={`
  d.synth("sine").root("c4").scale("maj").note([[0,3,5]],[[4,6,-1]]).push();
  d.synth("tri").root("c5").scale("maj").note([,0,,3,,5,,7],[4,6,8,10]).fil("lp", 1600).push();
`}
/>

### `offset(...patterns)`

Conceptually, this mathod functions similarly to the `note` method described above. Instead of expecting MIDI notes, though, it expects numbers between `0` and `1`, which defines the point in the sample where playback should begin. For example, passing in `0.5` will begin playing halfway through the duration of the sample.

| Argument    | Type                                   | Required |
| ----------- | -------------------------------------- | -------- |
| ...patterns | `number` \| `number[]` \| `number[][]` | `true`   |

```js
d.sample("my_loop").offset([0.25, 0, 0.625, 0.5]).push();
```

## Pattern Modifiers

Unlike the pattern setter methods above, these methods don't define what to play — they reshape when and how the pattern plays. They can be stacked in any order and chained freely with pattern setters.

### `xox(...patterns)`

Enables [X0X-style step sequencing](https://www.youtube.com/watch?v=YvOWB2UsbJs&t=55s), as made famouse by drum machines like the TR-808. You can define patterns in either of two ways: a string, using `x` to denote hits and any other character to denote rests; or an array, using `1` (or a truthy value) to denote hits and `0` (or a falsy value) to denote rests. Passing multiple strings or arrays creates a series of patterns

| Argument    | Type                   | Required |
| ----------- | ---------------------- | -------- |
| ...patterns | `string` \| `number[]` | `true`   |

<CodeEditor
  client:load
  code={`
d.sample("bd").xox("x . . . x . . .").push();
d.sample("hh").xox("x x x x x - x x").push();
d.sample("sd").xox([0, 0, 1, 0, 0, 0, 1, 0]).push();`}
/>

### `.euclid(pulses, steps, rotation)`

Generates a [Euclidean rhythm](https://en.wikipedia.org/wiki/Euclidean_rhythm) using an algorithm that evenly distributes a number of pulses across a specified number of steps — a classic technique for generating polyrhythmic patterns.

Passing an optional rotation argument shifts the pattern to the left (positive) or right (negative) by that many steps. Passing an array of values for either the pulses or rotation arguments creates multiple bars.

| Argument | Type                 | Required |
| -------- | -------------------- | -------- |
| pulses   | `number \| number[]` | `true`   |
| steps    | `number`             | `true`   |
| rotation | `number \| number[]` | `false`  |

<CodeEditor
  client:load
  code={`
// 3 hits over 8 steps, rotated 3 steps to the left
d.sample("bd:3").euclid(3,8,3).push();
// 2 hits over 8 steps, rotated 2 to the left 
d.sample("sd").euclid(2,8,2).gain(0.625).push(); 
// [11, 13] hits over 16 steps, rotated 5 steps to the right
d.sample("hh").euclid([11,13],16,-5).gain([0.5,0.25]).push();`}
/>

### `hex(...patterns)`

Express a pattern as a hexadecimal, which can be passed as a [hexadecimal integer literal](https://www.ibm.com/docs/en/i/7.4.0?topic=literals-hexadecimal-integer) or a string. Each base 16 hex digit encodes 4 steps as a [base 2 binary number](https://mathmonks.com/number-system/hexadecimal-to-binary).

```js
d.sample("bd").hex("f").push(); // string: 4 on the floor kick
d.sample("hh").hex(0xffff).push(); // hex literal: 16th note hi hat
```

| Argument    | Type                              | Required |
| ----------- | --------------------------------- | -------- |
| ...patterns | `hexadecimal literal` \| `string` | `true`   |

<CodeEditor
  client:load
  code={`
d.sample("bd").hex(0xF).push();
d.sample("hh").hex(0xAAABAABA).push(); 
d.sample("oh").hex(0x55).push();
d.sample("sd").hex(0x5).push();`}
/>

### `sequence(steps, ...pulses)`

Think of this as placing pins on a grid. You declare the total number of steps, then list the positions where hits should land — either as individual positions (one per bar) or as arrays of positions (multiple hits in a single bar). Where `euclid` distributes hits mathematically, `sequence` puts you in full spatial control.

| Argument  | Type                 | Required |
| --------- | -------------------- | -------- |
| steps     | `number`             | `true`   |
| ...pulses | `number \| number[]` | `true`   |

<CodeEditor
  client:load
  code={`
d.sample("bd").sequence(8, [0, 4]).push();
d.sample("sd").sequence(8, [2, 6]).push();
d.sample("hh").sequence(16, [0,2,4,6,7,8,10,12,14]).push();`}
/>

### `arrange(...patterns)`

Build longer, evolving sequences by stitching together multiple patterns, each one playing for a specified number of bars before handing off to the next. It's the simplest form of song structure — a way to write patterns that progress over time without relying on randomness, probability, or automation. Each argument is a tuple of `[numBars, pattern]`.

| Argument    | Type                                 | Required |
| ----------- | ------------------------------------ | -------- |
| ...patterns | `[number, (number \| number[])[]][]` | `true`   |

<CodeEditor
  client:load
  code={`
const lead_arr = [
    [3, [[-5,-1],,0,,-5,,-4,0,,0,5,,,,,,]], 
    [1, [,,[-3,-1],,-3,-2,,0,,0,7,,0,,4,,]]
]

d.synth("saw").arrange(...lead_arr).root("c4").scale("maj")
\ \ .adsr(0,1,0.5,0.125)
\ \ .fil("lp", d.env(400,1600).adsr(0,0.5,0.2,0.1))
\ \ .fx(d.reverb(0.2), d.delay(0.125,0.25)).push()`}
/>

### `fast(multiplier)`

Crams more repetitions of the pattern into each bar by compressing it in time. Passing `2` doubles the tempo of the pattern — fitting two full repetitions into the space of one bar. Passing a value less than `1` is equivalent to calling `slow()`.

| Argument   | Type     | Required |
| ---------- | -------- | -------- |
| multiplier | `number` | `true`   |

<CodeEditor
  client:load
  code={`
d.synth("saw").root("a4").scale("min").note([0,2,4,6]).fast(4)
\ \ .fil("lp",d.lfo(400,1200,0.25).off(0.5).norm())
\ \ .fx(d.reverb(0.5),d.delay(0.125,0.25))
\ \ .push()`}
/>

### `slow(multiplier)`

Spreads the pattern across more bars, stretching it out in time. Passing `2` halves the playback speed, so one pass through the pattern takes two bars instead of one. Useful for long, evolving phrases or for creating a pattern that feels like it's drifting behind the beat. Passing a value less than `1` is equivalent to calling `fast()`.

| Argument   | Type     | Required |
| ---------- | -------- | -------- |
| multiplier | `number` | `true`   |

<CodeEditor
  client:load
  code={`
d.synth("sine").root("c4").scale("maj")
  .note([0, 2, 4, 5, 7]).slow(2).push();
d.sample("bd").hex(0xf).slow(4).push();`}
/>

### `stretch(multiplier)`

Inserts rests between each existing hit, expanding the rhythmic spacing while keeping the pattern within the same number of bars. Unlike `slow()`, which stretches across bar boundaries, `stretch()` operates within the current bar — hits stay in place relative to the bar length and silence fills in between them. Great for sparse, spacious grooves.

| Argument   | Type     | Required |
| ---------- | -------- | -------- |
| multiplier | `number` | `true`   |

<CodeEditor
  client:load
  code={`
d.sample("bd").hex(0xf).stretch(4).push();
d.sample("sd").euclid(2, 8).stretch(2).push();`}
/>

### `reverse()`

Flips the entire pattern back-to-front — reversing both the order of bars and the steps within each bar. A melodic run becomes a descent, a forward-driving groove becomes its mirror image. Simple but surprisingly effective for creating variation without writing new material.

<CodeEditor
  client:load
  code={`
d.synth("sine").root("c4").scale("maj")
\ \ .note([0, 2, 4, 7]).reverse().push();

d.sample("hh").hex(0xd9).reverse().push();`}
/>

### `legato(should?)`

Ties each note into the next one, extending its duration until the following non-null step rather than cutting off at the end of its allocated step. Rests become connective tissue rather than silence — the note just keeps sounding through them. Essential for smooth melodic lines, sustained pads, and any passage where you want phrasing instead of staccato chops.

| Argument | Type      | Required | Default |
| -------- | --------- | -------- | ------- |
| should   | `boolean` | `false`  | `true`  |

<CodeEditor
  client:load
  code={`
// Without legato: short stabs
d.synth("sine").root("c4").scale("maj").note([0, null, 4, null]).push();
// With legato: each note sustains into the rest
d.synth("sine").root("c4").scale("maj").note([0, null, 4, null]).legato().push();`}
/>
